# Substack-as-Code

Writing samples by Matthew Manno, Ph.D., presented in docs-as-code formats.

This repository demonstrates the application of technical documentation 
markup languages — RST, Markdown, and DITA XML — to analytical and 
creative non-fiction writing. Each file is a version of a piece published 
or drafted for [mvclair.substack.com](https://mvclair.substack.com).

## Files

- **Destiny 2.rst** — Analytical essay on *Destiny 2* as a science fiction 
  text, formatted in reStructuredText (RST)
- **On Publishing (on GitHub).md** — Reflections on docs-as-code workflows 
  and publishing writing in version-controlled environments, formatted in Markdown

## Purpose

Technical writers work in many content formats. This repository shows that 
docs-as-code workflows — the same tools used in enterprise documentation 
pipelines — apply to any kind of writing, not just software documentation.
